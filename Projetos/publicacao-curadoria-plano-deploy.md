# Publicação da Página de Curadoria — Plano de Deploy

## Objetivo
Colocar a página de curadoria online com o menor atrito possível, para que os pins passem a ter um destino monetizável.

## Caminho operacional recomendado agora
**GitHub Pages** como primeira opção prática, porque:
- custo zero
- aceita página estática simples
- deploy rápido
- permite atualizar links depois sem reconstrução complexa

## O que já deixei pronto
- página HTML base pronta em `site/curadoria/index.html`
- placeholders dos links organizados
- nota de disclosure preparada
- checklist de preenchimento preparada em `Textos/afiliado-primeiros-links-template.md`

## O que ainda depende de ação externa
### Bloco A — Hospedagem
Você precisará:
1. entrar em uma conta GitHub
2. criar ou usar um repositório para a página
3. publicar via GitHub Pages

### Bloco B — Monetização
Você precisará:
1. entrar na conta/programa de afiliado
2. gerar os primeiros links reais
3. me passar os links ou inseri-los

## Passagem de bastão mínima
Se você quiser destravar com o menor esforço possível, a ordem é:
1. publicar `site/curadoria/index.html`
2. gerar 3 links reais de afiliado
3. substituir os placeholders
4. atualizar o destino do pin 1

## Onde eu paro se houver limitação técnica
Eu paro quando a tarefa exigir:
- login
- captcha
- 2FA
- criação/validação de conta externa
- aceite de termos
- dados bancários/fiscais

## Próxima ação interna sugerida
Depois da publicação, eu preparo imediatamente:
- versão com links reais inseridos
- mapeamento pin -> seção -> produto -> link
- sequência monetizável do pin 2 ao pin 6
